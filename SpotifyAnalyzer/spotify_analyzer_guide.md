## 🔑 Решение проблемы с плейлистами (Rate Limit 429)

Поскольку общий API-ключ приложения используется всеми копиями анализатора, Spotify временно ограничил к нему доступ (Rate Limit 429). 
Чтобы **вернуть отображение всех плейлистов на сайте за 1 минуту** и больше никогда не сталкиваться с этой проблемой, настройте свои личные ключи разработчика. Это абсолютно бесплатно, безопасно и делается в 3 простых шага:

### Шаг 1: Получите личные Client ID и Client Secret
1. Перейдите на официальный портал разработчиков Spotify: [Spotify Developer Dashboard](https://developer.spotify.com/dashboard) и войдите под своей обычной учетной записью Spotify.
2. Нажмите кнопку **Create App** (Создать приложение) в правом верхнем углу.
3. Заполните форму:
   * **App name**: `Spotify Analyzer`
   * **App description**: `My personal music dashboard`
   * **Redirect URI**: `http://127.0.0.1:8080/` *(Обязательно введите этот адрес точно, включая слэш в конце)*
   * Отметьте галочку **Web API** в списке APIs.
4. Нажмите кнопку **Save** (Сохранить).
5. На странице созданного приложения нажмите кнопку **Settings** (Настройки) в верхнем правом углу.
6. Вы увидите свой личный **Client ID**. Нажмите кнопку **View Client Secret**, чтобы отобразить секретный ключ.

### Шаг 2: Создайте файл с ключами в папке проекта
1. В папке `C:\Users\svato\SpotifyAnalyzer` создайте текстовый файл и назовите его `spotify_credentials.json` (или переименуйте `spotify_credentials.json.example`).
2. Вставьте в него ваши скопированные ключи в следующем формате:
   ```json
   {
       "client_id": "ВАШ_ЛИЧНЫЙ_CLIENT_ID",
       "client_secret": "ВАШ_ЛИЧНЫЙ_CLIENT_SECRET"
   }
   ```
3. Сохраните файл.

### Шаг 3: Запустите скрипт
Откройте PowerShell в папке проекта и запустите:
```powershell
python classifier.py
```
Скрипт автоматически обнаружит ваши личные ключи, просканирует все ваши плейлисты на полной скорости без каких-либо лимитов, привяжет их к вашим песням на сайте и **сохранит полную базу данных в локальный файл `playlists_cache.json`**. 

> [!TIP]
> После первого успешного запуска с личными ключами плейлисты сохранятся в файл `playlists_cache.json`. В дальнейшем скрипт будет загружать их мгновенно из этого файла, даже если у вас не будет интернета или авторизация Spotify будет временно отключена!

## 🤖 Интеграция ИИ-Аудитора и Синхронизация Плейлистов

### 🔍 Как это работает в фоновом режиме:
1. При запуске `classifier.py` скрипт пытается инициализировать Spotify API.
2. Он загружает полный список плейлистов пользователя (`sp.current_user_playlists`).
3. Применяется **интеллектуальная функция фильтрации** `is_genre_playlist(playlist_name)`:
   * **Служебные списки** (*Все песни, На потом, Рекомендации, Фильмы, Тик ток, Каверы, Концерты, Лайвы, Просто чтобы отображалось, Баня*) игнорируются.
   * **Персональные плейлисты исполнителей** (*RHCP, Slipknot, Metallica, Gone.Fludd, Noize MC, Saluki* и т.д.) отфильтровываются.
   * **Временные папки по годам** (*Мой 2023, Мой 2024, Мой 2025*) отсекаются.
   * **Устаревшие жанры** по требованию пользователя (*Рок, Эмбиент, Русский поп детства, Блюз/Джаз/Соул, ENG Реп, Рок невошедшее*) полностью исключаются.
4. В памяти создается высокоэффективная хэш-карта соответствий `(Исполнитель, Название) -> Список плейлистов`.
5. Во время парсинга CSV-файла каждый трек сопоставляется с этой картой, и плейлисты прикрепляются к метаданным трека!

### ⚡ Экстремальное ускорение API (в 2 раза!):
За счет пре-фильтрации не-жанровых и служебных папок до начала фазы загрузки треков, количество плейлистов, опрашиваемых по API, сократилось со 171 до **86 плейлистов**! Это снизило нагрузку на API Spotify и время выполнения ровно в 2 раза: всего **33 секунды**!

## 🎨 Премиальные Визуальные Улучшения и Дизайн

### 🏷️ Отображение плейлистов в UI
В интерактивные карточки треков интегрирован новый блок метаданных:
* На каждой карточке выводится изящный маркер: `В плейлистах: [Теги плейлистов]`.
* Плейлисты, в которые добавлен данный трек, отображаются в виде компактных неоново-зеленых тегов-пилюль.
* Если трек пока не добавлен ни в один плейлист, отображается аккуратная приглушенная метка `Нет`.

### 🌈 Динамические HSL-Градиенты Обложек
Так как локальный CSV-файл с Chosic не содержит прямых ссылок на изображения обложек альбомов, мы внедрили **интеллектуальный генератор визуалов** на стороне фронтенда. 
Если у трека нет картинки обложки:
1. Хэш-функция преобразует имя исполнителя и название трека в уникальное числовое значение.
2. Это значение проецируется на спектр HSL (цветовой тон), генерируя **абсолютно уникальный, глубокий и сочный неоновый градиент** для каждой песни.
3. В центре градиента отображается первая буква имени артиста в современном геометрическом шрифте `Comfortaa`.
4. Вся плитка реагирует на наведение курсора мягким кинетическим зумом `scale(1.1)`.

### 🎹 Гармоническое Сведение (Camelot Keys)
Для диджеев и ценителей качественных переходов мы реализовали поддержку **Camelot Wheel**:
* При парсинге CSV Camelot-тональность считывается напрямую.
* При загрузке через Spotify Live API тональность **рассчитывается математически** на основе высоты звука (`pitch key`) и лада (`mode`):
  * **Мажорные клавиши** проецируются на `B`-диапазон (например, `8B` для C-major).
  * **Минорные клавиши** проецируются на `A`-диапазон (например, `9A` для E-minor).
* Тональности выводятся в формате `E Минор [9A]`, позволяя вам мгновенно оценивать возможности для гармонического микширования прямо в карточках треков.

## 📝 Список последних изменений (Changelog)

### 🛡️ Бесшовное закрытие скролла, защита от Rate Limit и личный API-кабинет (Версия 1.7.8)
* **🔒 Абсолютная заморозка скролла (Cross-Browser Scroll Lock)**: Исправлена проблема с прокруткой заднего фона на некоторых версиях Chrome и Safari. Теперь при открытии модального окна класс `modal-active` применяется не только к `body`, но и к корневому элементу `html`, а в CSS прописана жесткая связка `overflow: hidden !important` и `height: 100vh`, полностью блокирующая любые перемещения фонового содержимого.
* **⚡ Иммунитет к Rate Limit (Semicolon Parser Fix)**: Полностью переработана функция восстановления кэша плейлистов `extract_mapping_from_existing_html`. Логика прямого поиска символа `;` заменена на высоконадежный встроенный декодер `json.JSONDecoder().raw_decode()`. Теперь парсер не боится точек с запятой в названиях песен, альбомах или жанрах, гарантируя 100% восстановление плейлистов из существующего отчета даже при полной блокировке Spotify API.
* **🔑 Личный API-кабинет (`spotify_credentials.json`)**: Внедрена поддержка персональных учетных данных разработчика. Если общий ключ приложения превышает лимиты запросов (Rate Limit 429), пользователь может создать простой файл `spotify_credentials.json` в корне проекта. Скрипт автоматически подхватит личные ключи, просканирует всю медиатеку без ограничений и сохранит ее в локальный файл кэша `playlists_cache.json`.

### 🖼️ Микро-окошко с деталями песни (Версия 1.7.7)
* **Премиальный Details Modal**: При клике на карточку песни плавно всплывает красивое микро-окошко с полной подноготной трека. Задний фон элегантно размывается (`backdrop-filter: blur(15px)`).
* **Шкалы «Музыкального ДНК»**: В окошке выводятся динамические, переливающие Progress Bars для Danceability, Energy, Valence, Acousticness и Instrumentalness, которые красиво заполняются при открытии.
* **Полная сводка характеристик**: Отображаются Camelot-тональность, BPM, Вайб, Популярность, Альбом и все плейлисты, куда добавлена песня.
* **Защита кликов**: Нажатия на кнопку копирования Gemini, превью-плеер или ссылку Spotify перехватываются и не мешают вызову модального окна.

### 🔗 Кликабельные заголовки треков (Версия 1.7.6)
* **Кликабельные названия песен**: Название каждого трека теперь является красивой ссылкой. Клик по нему открывает песню в новом окне/вкладке браузера для детального просмотра напрямую на Spotify.
* **Премиальный ховер-эффект**: Ссылка плавно окрашивается в неоново-зеленый цвет Spotify Green при наведении и получает деликатное подчеркивание.
* **Fail-Fast защита при лимитах API**: Spotipy настроен на моментальный выход (`retries=0`, `status_retries=0`), а HTTP 429 Rate Limit перехватывается, предотвращая зависания и запуская быструю офлайн-генерацию с уже считанными данными.

### 🧠 Умные Жанровые Плейлисты (Версия 1.7.5)
* **Интеллектуальная фильтрация**: Скрипт сканирует все 171 плейлист пользователя и отбрасывает не-жанровые списки (артистов, Favorites, папки годов, служебные) и исключенные старые жанры.
* **Экстремальный прирост скорости**: Время фоновой синхронизации сократилось в два раза — программа отрабатывает всего за **33 секунды** вместо 1 мин 10 сек!
* **Удаление старых жанров**: По требованию пользователя, плейлисты *Рок, Эмбиент, Русский поп детства, Блюз/Джаз/Соул, ENG Реп, Рок невошедшее* полностью удалены из результатов аудита.
